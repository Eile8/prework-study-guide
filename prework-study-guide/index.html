HTML

  • HTML stands for Hypertext Markup Language that is used to create the structure of a webpage.
  • The head element contains information about the webpage.
  • The metadata contains information about the page that is used by the browser.
  • Both of the elements cannot be seen by the user.
  • The boy element represents the visible content shown to the user.
  • Semantic HTML elements describes its content and the role it plays on a webpage.
  • It is used to ensure webpage is accessible to the widest variety of user such as someone who needs screen reader feature.

CSS

  • css helps to add styles in the webpage such as color, shape, and fonts.
  • inline CSS: it applies to a specific HTML tag; style a particular page element only.
  • internal style: the rules are in the head section of HTML file
  • external style: allows developers to keep all their css rules in a separate file then link the file to HTML's head section.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git: a version control systemthat keeps track and manage each version of projects.
  • Repositories, AKA repos, are folders that store:
  • Files that contain code
  • Files that contain other information about the code, like a README file.
  • The version history for all of the files in repository.
  • One key feature of the repositories is it allows developers to share and collaborate a project much easier.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it
